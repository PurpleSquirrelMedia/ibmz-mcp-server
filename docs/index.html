Features

HSM-Backed Key Management

Enterprise-grade encryption with FIPS 140-2 Level 3 certified Hardware Security Modules. Keys never leave the HSM hardware.

z/OS Connect Integration

REST APIs to mainframe programs. Call CICS transactions, IMS programs, and batch jobs directly from Claude.

Envelope Encryption

Industry-standard envelope encryption pattern. Root keys wrap data encryption keys for secure key hierarchy management.

JSON to COBOL Mapping

z/OS Connect automatically maps JSON payloads to COBOL copybooks. Modernize mainframe access without changing legacy code.

Available Tools

Key Protect (HSM Key Management)

Tool Description
key_protect_list_keys List all encryption keys in your Key Protect instance
key_protect_create_key Create a new root key (KEK) or standard key (DEK)
key_protect_get_key Get metadata and details of a specific key
key_protect_wrap_key Wrap (encrypt) a DEK with a root key for envelope encryption
key_protect_unwrap_key Unwrap (decrypt) a wrapped DEK using a root key
key_protect_rotate_key Rotate a root key while maintaining access to old versions
key_protect_delete_key Delete a key (WARNING: irreversible)
key_protect_get_key_policies Get policies (rotation, dual auth) applied to a key

z/OS Connect (Mainframe Integration)

Tool Description
zos_connect_list_services List all available mainframe services (REST APIs)
zos_connect_get_service Get service details and OpenAPI specification
zos_connect_call_service Call a mainframe service (CICS, IMS, batch)
zos_connect_list_apis List API requester configurations (outbound calls)
zos_connect_health Check z/OS Connect server health status

Quick Start

1

Install Dependencies

Clone the repository and install npm packages

# Clone the repository
git clone https://github.com/PurpleSquirrelMedia/ibmz-mcp-server.git
cd ibmz-mcp-server

# Install dependencies
npm install
2

Configure Claude Code

Add the MCP server to your ~/.claude.json configuration

{
  "mcpServers": {
    "ibmz": {
      "type": "stdio",
      "command": "node",
      "args": ["/path/to/ibmz-mcp-server/index.js"],
      "env": {
        "IBM_CLOUD_API_KEY": "your-api-key",
        "KEY_PROTECT_INSTANCE_ID": "your-instance-id"
      }
    }
  }
}
3

Restart Claude Code

The IBM Z tools will be available after restart

# Restart Claude Code to load the new MCP server
claude

# Example usage
User: List my encryption keys in Key Protect
Claude: [Uses key_protect_list_keys tool]

Key Concepts

Envelope Encryption

Industry-standard pattern for managing encryption at scale:

Root Keys (KEK)

  • Stored in HSM hardware - never exported
  • Used to wrap (encrypt) data encryption keys
  • Can be rotated without re-encrypting data

Data Encryption Keys (DEK)

  • Used to encrypt actual data
  • Wrapped by root keys for protection
  • Stored alongside encrypted data

z/OS Connect

RESTful access to mainframe programs:

Supported Subsystems

  • CICS - Online transaction processing
  • IMS - Hierarchical database and transactions
  • Batch - Scheduled job processing
  • Db2 - Relational database access

Automatic Mapping

  • JSON to COBOL copybook conversion
  • OpenAPI specification generation
  • No changes to legacy code required
